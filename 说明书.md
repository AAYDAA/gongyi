# 爱心公益平台 说明书

## 1. 项目简介

本项目为一个公益平台示例系统，提供志愿活动报名、资源需求发布与认领、公益项目捐款、公益资讯展示、公益组织认证、志愿者申请审核等功能，并提供管理员审核与管理入口。

项目采用前后端分离：

- 后端：Spring Boot 3.x + Spring Security + JWT + MyBatis + PageHelper
- 前端：Vite + Vue3 + TypeScript + Vue Router + Pinia + Element Plus + TailwindCSS

## 4. 快速启动

### 4.1 后端启动

1) 修改后端配置文件：[application.yml](file:///d:/A.AA.Leetcode/A.%E6%AF%95%E8%AE%BE/%E5%85%AC%E7%9B%8A%E6%85%88%E5%96%84%E7%B3%BB%E7%BB%9F/backend/src/main/resources/application.yml)

- `spring.datasource.url/username/password` 改为本机 MySQL 配置
- `app.jwt.secret` 修改为一个足够长的随机字符串
- （重要）修改默认管理员账号初始化配置 `app.init-admin.*`（首次启动会自动创建）

2) 启动后端：

```bash
mvn -f backend/pom.xml spring-boot:run
```

默认服务地址：`http://localhost:8080`

### 4.2 前端启动

```bash
cd frontend
npm install
npm run dev
```

默认访问地址：`http://localhost:5173`

- `GET /api/activities` 分页列表

## 7.1 管理员如何登录

项目默认提供"启动时自动创建管理员账号"的机制（仅当数据库内不存在同名账号时才创建）。

配置位置：

- [application.yml](file:///d:/A.AA.Leetcode/A.%E6%AF%95%E8%AE%BE/%E5%85%AC%E7%9B%8A%E6%85%88%E5%96%84%E7%B3%BB%E7%BB%9F/backend/src/main/resources/application.yml) 的 `app.init-admin.*`
- 初始化逻辑： [AdminInitializer.java](file:///d:/A.AA.Leetcode/A.%E6%AF%95%E8%AE%BE/%E5%85%AC%E7%9B%8A%E6%85%88%E5%96%84%E7%B3%BB%E7%BB%9F/backend/src/main/java/com/gongyi/platform/config/AdminInitializer.java)

默认值（建议你启动前就改掉密码）：

- 用户名：`admin`
- 密码：`admin123456`

登录步骤：

1) 启动后端（确保已连上 MySQL）
2) 打开前端 `http://localhost:5173/login`
3) 使用上面的管理员账号登录
4) 登录后右上角头像菜单可进入"管理后台"

## 7.2 如何添加公益项目

**方式 A：管理员在前端后台新建（推荐）**

1) 管理员登录
2) 访问 `http://localhost:5173/admin/project-manage`
3) 点击"新建"并填写标题、封面图、摘要、目标金额、状态
4) 若要开放捐款，把项目状态设为 `ACTIVE`

**方式 B：公益组织创建自己的项目**

1) 注册账号时选择角色为 `ORGANIZATION`
2) 登录后进入"组织中心"完善资料
3) 进入 `http://localhost:5173/org/projects` 创建项目

**方式 C：直接调用接口**

- `POST /api/projects`（需要 ORGANIZATION 或 ADMIN 权限）
- `PUT /api/projects/{id}` 修改项目状态为 `ACTIVE` 后可捐款

## 7.3 如何新增志愿者活动 & 获取报名资料

**发布入口（公益组织/管理员）**

后端支持创建志愿活动：

- `POST /api/activities`

前端入口（组织账号登录后）：

- 右上角头像下拉 → "活动管理"
- 或直接访问：`http://localhost:5173/org/activities`

创建完成后，如果要允许志愿者报名，需要把活动状态设为 `RECRUITING`（招募中）。

**报名资料获取（活动发布者/管理员）**

后端接口：

- `GET /api/activities/{id}/applications`

前端入口：

- 在"我发布的志愿活动"列表中点击某条活动的"报名资料"按钮即可查看分页列表。
