**问题一：系统的整体架构是怎样的？**

答：采用前后端分离架构。前端 Vue3 + TypeScript + Element Plus，后端 Spring Boot 3 + MyBatis + MySQL，认证使用 Spring Security + JWT 无状态令牌。前后端通过 RESTful API 通信，前端负责页面渲染，后端负责业务逻辑与数据持久化。

**问题二：如何实现四种角色的权限控制？**

答：系统定义 ADMIN、ORGANIZATION、VOLUNTEER、USER 四种角色，使用 `@PreAuthorize` 注解实现方法级权限拦截。例如报名志愿活动需 VOLUNTEER 角色，发布项目需 ORGANIZATION 或 ADMIN。公开查询接口允许匿名访问，写操作需 JWT 令牌校验。注册默认获得 USER 角色，提交志愿者申请并通过审核后升级为 VOLUNTEER。

**问题三：数据库设计了哪些核心表？如何保证数据一致性？**

答：9 张表：users、volunteer_applications、organization_profiles、activities、activity_applications、charity_projects、donation_records、resources、resource_donations。表间通过主键外键关联，业务操作使用 `@Transactional` 保证事务一致性，如捐款时同步更新项目已筹金额，任一操作失败整体回滚。

**问题四：志愿者报名活动的完整流程和身份校验机制？**

答：流程：注册（USER）→ 提交志愿者申请 → 管理员审核通过（升级 VOLUNTEER）→ 浏览活动 → 报名 → 发布方审核。校验采用双重机制：Controller 层 `@PreAuthorize("hasRole('VOLUNTEER')")` 拦截；SecurityUtils 从 JWT 提取当前用户 ID 绑定报名记录，防止伪造身份。

**问题五：系统还有哪些可优化的方向？**

答：①引入 Redis 缓存热点数据；②接入真实支付接口；③增加消息通知模块（短信/邮件）；④引入 Elasticsearch 优化全文检索；⑤补充单元测试与 E2E 测试。
